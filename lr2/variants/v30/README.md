# Варіант 30 — Повний приклад виконаної лабораторної

Це **повністю розв'язаний варіант 30** лабораторної роботи №2. Використовуйте як зразок разом з `demo/`.

## Різниця між demo/ та v30/

| | demo/ | v30/ |
|---|-------|------|
| Що це | Інтерактивний приклад від викладача | Повне рішення варіанту 30 |
| Дані | Демонстраційні (не з жодного варіанту) | Точно з v30.md |
| Для чого | Зрозуміти як працює кожне завдання | Побачити як виглядає здана робота |

## Особливості варіанту 30 (група C)

| Завдання | Відмінність від demo/ |
|----------|----------------------|
| 2 Sort | `rsort` — зворотний алфавітний порядок (demo: `sort`) |
| 4 Dates | + день тижня українською (demo: тільки дні) |
| 5 Password | Пароль без підрядка логіну ≥3 символи (demo: звичайна генерація) |
| 6 Duplicates | Мода — найчастіший елемент (demo: дублікати) |
| 8 Arrays | `array_intersect` + sort ↑ (demo: `merge` + `unique` + sort) |
| 9 Assoc | ім'я => зарплата (demo: ім'я => вік) |

## Запуск

```bash
php -S localhost:8000
```

Відкрийте: `http://localhost:8000/lr2/variants/v30/`

## Як використовувати як шаблон

1. Скопіюйте цю папку: `cp -r v30 vN` (де N — ваш номер варіанту)
2. Відкрийте ваш `vN.md` — там ваші конкретні дані
3. Замініть дані в кожному файлі на свої з `vN.md`
4. Перевірте: `http://localhost:8000/lr2/variants/vN/`

> **Не змінюйте цю папку** — вона доступна всім студентам як приклад.
