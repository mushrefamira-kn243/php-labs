## Завдання 1: Структура MVC

Створити скелет веб-додатку з MVC на тему **"Комп'ютерний клуб"**. Сторінки:
- Головна (статична, з описом клубу: тарифи, обладнання, турніри, графік роботи)
- Форма реєстрації гравця (з валідацією полів)
- Сторінка перегляду GET та POST параметрів
- Єдина точка входу (`index.php`)

Очікувана структура:
```
pc-club/
├── classes/ (Application, Controller, PageController, Request, Router, View, PageView)
├── config/ (init.php)
├── controllers/ (IndexController, RegformController, ReqviewController, SettingsController)
├── views/
│   ├── layout/ (header.php, footer.php)
│   ├── index/ (main.php)
│   ├── regform/ (form.php, done.php)
│   ├── reqview/ (showrequest.php)
│   └── settings/ (color.php, greeting.php)
├── css/ (style.css)
├── index.php
└── .htaccess
```

## Завдання 2: Модулі

### 2.1 Модуль обробки форми

**Тип форми:** T5 — Калькулятор для розрахунку вартості сесії

**Поля форми:**
- Число 1 (text) — перше число
- Число 2 (text) — друге число
- Функція (select) — вибір операції:
  - Сума (+)
  - Різниця (-)
  - Множення (*)
  - Ділення (/)
  - Степінь (^)
  - Корінь квадратний (√)
- Результат (text) — відповідь користувача

**Правила перевірки:**
- Сума, різниця, множення — будь-які числа
- Ділення — Число 2 ≠ 0, інакше → "Ділення на нуль неможливе"
- Степінь — будь-які числа (Число 1 ^ Число 2)
- Корінь квадратний — Число 1 > 0, Число 2 не використовується. Якщо Число 1 ≤ 0 → "Корінь з від'ємного числа неможливий"
- Обчислити правильний результат і порівняти з введеним. Якщо не збігається → "Результат невірний. Правильна відповідь: X"

### 2.2 Модуль сесій

На окремій сторінці — вибір кольору фону:
- Збереження в `$_SESSION['bg_color']`
- Застосування на всіх сторінках сайту

### 2.3 Модуль Cookie

- Ім'я (text) + Стать (radio: ЧОЛ/ЖІН)
- Cookie на 30 днів
- "Вітаємо Вас, пане/пані ІМ'Я"
